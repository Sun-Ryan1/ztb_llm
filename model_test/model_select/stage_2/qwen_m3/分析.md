  - RAG prompt更长，生成更耗时

### 2. **主要性能瓶颈**

#### 🔴 **最耗时的环节：**
1. **逐条RAG推理**（场景2）
   - 每个问题独立处理：检索 → 构建prompt → 生成
   - 无批量处理，GPU利用率低
   - RAG prompt长达1000+ token，生成速度慢

2. **检索环节**
   - `enhanced_retrieval`每次都要计算问题向量 + FAISS搜索 + 关键词匹配
   - 虽然FAISS搜索快，但关键词匹配和向量编码仍有开销

3. **模型生成速度**
   - Qwen-7B在消费级GPU（如RTX 3090/4090）上生成速度约20-40 token/秒
   - 520个问题 × 平均400 token ≈ 208,000 token
   - 仅生成时间就需约1.5-2小时

#### 🟡 **次要瓶颈：**
4. **数据加载与预处理**
   - JSON解析和文本处理
   - 评估函数中的实体提取和相似度计算

5. **I/O和日志开销**
   - 频繁的日志输出
   - 结果保存和文件操作
